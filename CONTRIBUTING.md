# Contributing

Dziękujemy za zainteresowanie projektem!

## Zgłaszanie zmian

1. Utwórz fork repozytorium i branch opisujący zmianę.
2. Wprowadź małą, jasno opisaną zmianę.
3. Uruchom lokalnie `python -m py_compile app.py`.
4. Otwórz pull request i opisz cel oraz sposób przetestowania zmiany.

Przed zgłoszeniem większej zmiany warto najpierw otworzyć issue, aby uzgodnić kierunek prac.

## Standard zmian

- Nie dodawaj sekretów ani pliku `.env` do repozytorium.
- Aktualizuj README, jeśli zmiana wpływa na uruchomienie projektu.
- Zachowuj prostotę i czytelność kodu.
